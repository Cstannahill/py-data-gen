# Example constraints file for Alpaca format

## Instruction Quality
- Instructions should be clear, specific, and actionable
- Vary complexity from simple to complex tasks
- Include diverse task types: analysis, creation, explanation, comparison, debugging, etc.
- Avoid overly broad or vague instructions

## Input Field Usage
- Use the input field for context when it adds value
- Examples: code snippets to analyze, text to summarize, data to process
- Input can be empty when the instruction is self-contained
- Keep inputs focused and relevant to the instruction

## Output Quality
- Outputs should fully address the instruction
- Include examples, explanations, or step-by-step guidance as appropriate
- Maintain consistent expertise level appropriate to the instruction
- Outputs should be thorough but not unnecessarily verbose
- Include caveats or edge cases when relevant

## Diversity Requirements
- Mix different domains: programming, writing, analysis, problem-solving, etc.
- Vary output lengths based on what the instruction requires
- Include both creative and analytical tasks
- Represent different difficulty levels
